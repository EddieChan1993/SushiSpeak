# SushiSpeak — Claude Notes

macOS SwiftUI app for English speaking practice. Swift Package Manager, no Xcode project.

## Build command (always return this after any change)

```bash
cd ~/code/SushiSpeak && ./build.sh
```

`build.sh`: kills running instance → `swift build -c release` → assembles `.app` bundle → code-signs → `open`.
Use `-d` for dev mode: debug build, skips bundling external tools.

## Project structure

```
Sources/SushiSpeak/
  SushiSpeakApp.swift        — @main entry point
  ContentView.swift          — UI: header, timer panel, format picker, waveform, recordings list
  AudioRecorder.swift        — AVAudioEngine recording + gain boost + RMS metering
  WhisperTranscriber.swift   — Whisper model management + async transcription
  Recording.swift            — data model (id, url, date, duration)
Assets/
  AppIcon.iconset/           — PNG sizes 16–1024 (generated by generate_icon.swift)
  AppIcon.icns               — compiled icon, bundled into Contents/Resources/
generate_icon.swift          — Swift script to redraw the icon (Core Graphics, no deps)
```

## Key decisions & rationale

### Audio recording: AVAudioEngine (not AVAudioRecorder)
`AVAudioRecorder` has no gain control. We use `AVAudioEngine` + `AVAudioMixerNode` with `outputVolume = 2.5` (~8 dB boost) so normal speaking volume records clearly.

### Audio format pipeline
Always record as M4A (AAC). After stopping, convert to user-selected format via bundled ffmpeg:
- **MP3**: `ffmpeg -y -i rec.m4a -q:a 2 rec.mp3`
- **WAV**: `ffmpeg -y -i rec.m4a rec.wav`
- **M4A**: no conversion, keep as-is

User selects format via picker in the timer panel; stored in `@AppStorage("audioFormat")`.

**Why not native MP3?** macOS AVFoundation has no MP3 encoder. ffmpeg is the only solid path.
**Critical**: extension must match format. `.mp3` + LinearPCM = silent failure.
**豆包 compatibility**: WAV rejected, AAC rejected. MP3 works.

### Whisper transcription (whisper-cpp)
Clicking the `waveform.and.mic` button in each recording row:
1. ffmpeg converts audio → 16kHz mono WAV (best accuracy for whisper)
2. `whisper-cli -m model.bin -f audio.wav -l auto --no-timestamps` runs
3. Transcript copied to clipboard; button shows green checkmark 2s

**Context awareness**: `--prompt` flag available; currently unused (pass previous transcript for continuity).
**Language**: `-l auto` handles Chinese, English, mixed.

**Models** (GGML format, stored in `~/Library/Application Support/SushiSpeak/models/`):
| Name | Size | Notes |
|------|------|-------|
| Tiny | ~75 MB | fastest, lowest accuracy |
| Base | ~142 MB | |
| Small | ~466 MB | **default** |
| Medium | ~1.5 GB | |
| Large V3 | ~3.1 GB | best accuracy |

Model picker in recordings panel header. Selecting an unavailable model triggers auto-download (streaming, 128 KB chunks, cancellable).

### Bundled external tools
Production build bundles into `Contents/MacOS/`:
- `ffmpeg` — audio format conversion, WAV pre-processing for Whisper
- `whisper-cli` — Whisper inference binary
- `libwhisper.1.dylib`, `libggml.0.dylib`, `libggml-base.0.dylib` — Whisper dylib dependencies
- `libggml-metal.so`, `libggml-blas.so`, `libggml-cpu-apple_m{1,2/3,4}.so` — ggml backend plugins

`build.sh` fixes `@rpath` on all dylibs with `install_name_tool`. `WhisperTranscriber` sets `GGML_BACKEND_PATH` + `DYLD_LIBRARY_PATH` env vars when launching the child process so bundled backends are found.

Dev mode (`./build.sh -d`): uses system Homebrew paths, skips bundling.

### App icon
Generated by `generate_icon.swift` (run manually, output committed to `Assets/`).
Design: flat-style sushi roll (top-down maki: nori → rice → salmon) + 3 white sound-wave arcs on a warm red-orange rounded square.
To regenerate: `cd ~/code/SushiSpeak && swift generate_icon.swift && iconutil -c icns Assets/AppIcon.iconset -o Assets/AppIcon.icns`

### Window title bar
`titleVisibility = .hidden` set via `WindowTitleHider` (`NSViewRepresentable`) applied as `.background()` on the root view. Keeps traffic-light buttons, removes the "SushiSpeak" title text. Header uses `NSApp.applicationIconImage` (the bundled `.icns`) resized to 26×26 pt.

### Timer settings persistence: @AppStorage
`selectedMinutes`, `selectedSeconds`, `audioFormatRaw`, `whisperModelRaw` all use `@AppStorage` (UserDefaults).

### Waveform: RMS metering with noise gate
Tap callback computes RMS per 4096-sample buffer. Noise gate: RMS < 0.012 → level = 0 (flat bars). Exponential smoothing (`old × 0.55 + new × 0.45`). `TimelineView(.animation)` drives 60 fps bar rendering.

### Completion alert
`NSSound(named: "Glass")?.play()` + `NSApp.requestUserAttention(.criticalRequest)`.
`UNUserNotificationCenter` and `NSUserNotificationCenter` both fail for non-sandboxed SPM apps.

### Focus lock during recording
`.disabled(isRunning).allowsHitTesting(!isRunning)` on hidden pickers. Opacity-only was insufficient — space bar still cycled keyboard focus.

### Height stability on Start
Waveform and duration picker share a `ZStack` at fixed `height: 48`, toggled by `.opacity`. `if/else` caused layout jump.

## Known issues / watch out

- `AVAudioEngine.inputNode.outputFormat` must only be queried after mic permission is granted — crash if called before.
- `booster.removeTap(onBus: 0)` must be called before `engine.stop()` — occasional EXC_BAD_ACCESS on rapid start/stop otherwise.
- `DYLD_LIBRARY_PATH` is set on the child Process environment; this works for non-system binaries but may be stripped by SIP in edge cases.
- Whisper `--prompt` context is not yet wired to previous transcripts in the UI.
- On first launch with no downloaded model, the transcribe button will fail silently — user must download a model first.
