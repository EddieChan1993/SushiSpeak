## Key decisions & rationale

### Audio recording: AVAudioEngine (not AVAudioRecorder)
`AVAudioRecorder` has no gain control. We use `AVAudioEngine` + `AVAudioMixerNode` with `outputVolume = 2.5` (~8 dB boost) so normal speaking volume records clearly.

### Audio format pipeline
Always record as M4A (AAC). After stopping, convert to user-selected format via bundled ffmpeg:
- **MP3**: `ffmpeg -y -i rec.m4a -q:a 2 rec.mp3`
- **WAV**: `ffmpeg -y -i rec.m4a rec.wav`
- **M4A**: no conversion, keep as-is

User selects format via picker in the timer panel; stored in `@AppStorage("audioFormat")`.

**Why not native MP3?** macOS AVFoundation has no MP3 encoder. ffmpeg is the only solid path.
**Critical**: extension must match format. `.mp3` + LinearPCM = silent failure.
**Ë±ÜÂåÖ compatibility**: WAV rejected, AAC rejected. MP3 works.

### Whisper transcription (whisper-cpp)
Clicking the `captions.bubble` button in each recording row (right-side action icons):
1. ffmpeg converts audio ‚Üí 16kHz mono WAV (best accuracy for whisper)
2. `whisper-cli -m model.bin -f audio.wav -l auto --no-timestamps` runs
3. Transcript shown in `TranscriptSheet` ‚Äî `NSTextView`-backed, fully selectable

**Context awareness**: `--prompt` flag available; currently unused (pass previous transcript for continuity).
**Language**: `-l auto` handles Chinese, English, mixed.

**Models** (GGML format, stored in `~/Library/Application Support/SushiSpeak/models/`):
| Name | File | Notes |
|------|------|-------|
| Tiny | `ggml-tiny.bin` | fastest, lowest accuracy |
| Base | `ggml-base.bin` | |
| Small | `ggml-small.bin` | **default** |
| Medium | `ggml-medium.bin` | |
| Large V3 | `ggml-large-v3.bin` | best accuracy |

Model picker lives in the **header bar** (right side, next to logo). No auto-download ‚Äî user imports `ggml-*.bin` files manually (download URL shown in üåê popover). Green dot on each picker item indicates model is present.

### Model import ‚Äî atomic, 3-phase

`importModelFile()` in ContentView opens `NSOpenPanel` with `allowsMultipleSelection = true`.

**Phase 1 (sync):** detect model from filename, reject unknown names, reject duplicates within the selection. Errors shown immediately; entire import aborted.

**Phase 2 (async):** `WhisperTranscriber.validateModelWorks(_:at:)` ‚Äî runs `whisper-cli` against a tiny 0.5 s silent WAV to confirm the model actually loads. Runs in `Task.detached` so the main thread is never blocked. UI shows a `ProgressView` spinner. Any subprocess failure surfaces the stderr text as a specific error. All models validated sequentially before any file is copied.

**Phase 3 (sync):** `importModel(_:from:)` copies each validated file to App Support. Picker switches to the last imported model.

File-name matching rules:
- Exact match: `ggml-small.bin` ‚Üí Small
- Fuzzy suffix: `ggml-small (1).bin` or `ggml-small copy.bin` ‚Üí Small (handles macOS duplicate-download names)
- Everything else ‚Üí error (e.g. `ggml-base.en.bin` is rejected)

No size check, no magic-bytes check ‚Äî actual execution is the ground truth.

### Bundled external tools
Production build bundles into `Contents/MacOS/`:
- `ffmpeg` ‚Äî audio format conversion, WAV pre-processing for Whisper
- `whisper-cli` ‚Äî Whisper inference binary
- `libwhisper.1.dylib`, `libggml.0.dylib`, `libggml-base.0.dylib` ‚Äî Whisper dylib dependencies
- `libggml-metal.so`, `libggml-blas.so`, `libggml-cpu-apple_m{1,2/3,4}.so` ‚Äî ggml backend plugins

`build.sh` fixes `@rpath` on all dylibs with `install_name_tool`. `WhisperTranscriber` sets `GGML_BACKEND_PATH` + `DYLD_LIBRARY_PATH` env vars when launching the child process so bundled backends are found.

Dev mode (`./build.sh -d`): uses system Homebrew paths, skips bundling.

**TCC microphone permission across rebuilds**: `build.sh` intentionally does NOT `rm -rf` the `.app` bundle itself ‚Äî it only wipes `Contents/` and rebuilds inside the existing `.app` directory. macOS TCC ties microphone permission to the bundle path; deleting and recreating the directory resets the inode and causes TCC to re-prompt. Keeping the `.app` directory alive preserves the grant across rebuilds.

### Window title bar
`titleVisibility = .hidden` set via `WindowTitleHider` (`NSViewRepresentable`) applied as `.background()` on the root view. Keeps traffic-light buttons, removes the "SushiSpeak" title text. Header uses `NSApp.applicationIconImage` (the bundled `.icns`) resized to 26√ó26 pt.

### Timer settings persistence: @AppStorage
`selectedMinutes`, `selectedSeconds`, `audioFormatRaw`, `whisperModelRaw`, `hideTimer` all use `@AppStorage` (UserDefaults).

### Hide-timer toggle
Eye button overlaid on the timer display (trailing edge). When active, shows `--:--` in dim gray instead of the live countdown ‚Äî layout is stable (same font/frame). Countdown continues normally. Useful for reducing anxiety during practice sessions.

### Waveform: RMS metering with noise gate
Tap callback computes RMS per 4096-sample buffer. Noise gate: RMS < 0.012 ‚Üí level = 0 (flat bars). Exponential smoothing (`old √ó 0.55 + new √ó 0.45`). `TimelineView(.animation)` drives 60 fps bar rendering.

### Timer accuracy ‚Äî wall-clock deadline + sync mic start
Two fixes to ensure recording duration matches the set time:

1. **Wall-clock deadline**: timer loop checks `endDate.timeIntervalSinceNow` every 500 ms instead of sleeping 1 s per tick. Avoids drift accumulation (~15 s over 5 min with the old approach).

2. **Sync mic start**: `startRecording(onStarted:)` skips the async `requestAccess` callback when permission is already granted (normal case after first launch) and calls `doStart()` synchronously. `endDate` is set inside `onStarted`, i.e. the moment the mic actually opens ‚Äî timer and recording are exactly aligned. On first launch (permission dialog), both still start together via the async path.

### Completion alert
`NSSound(named: "Glass")?.play()` + `NSApp.requestUserAttention(.criticalRequest)`.
`UNUserNotificationCenter` and `NSUserNotificationCenter` both fail for non-sandboxed SPM apps.

### Focus lock during recording
`.disabled(isRunning).allowsHitTesting(!isRunning)` on hidden pickers. Opacity-only was insufficient ‚Äî space bar still cycled keyboard focus.

### Height stability on Start
Waveform and duration picker share a `ZStack` at fixed `height: 48`, toggled by `.opacity`. `if/else` caused layout jump.

### Start/Stop button stays colored when window loses focus
`.borderedProminent` buttons go gray when the window is inactive by default. Fixed with `.environment(\.controlActiveState, .active)` on the button ‚Äî forces it to always render in the active state regardless of window focus.

### Button focus highlight (sheet close button)
The `xmark.circle.fill` close button in TranscriptSheet uses `.focusable(false)` to prevent it from capturing keyboard focus and showing the blue focus ring on sheet open.

## build.sh ‚Äî lessons learned

### DO NOT touch the dylib bundling logic unless absolutely necessary
The rpath-fixing section in `build.sh` (for `whisper-cli`, `libwhisper`, `libggml`) was carefully crafted and validated. Replacing it with "smarter" auto-discovery broke two things simultaneously:

1. **Never run `install_name_tool` on `.so` backend plugins** ‚Äî ggml loads them via `dlopen` with their own internal linking. Rewriting their paths causes `GGML_ASSERT(device) failed` at model load time.
2. **Never `strip` whisper-cli** ‚Äî `strip -x` removes local symbols that whisper-cli needs at runtime, causing transcription to return empty text silently.

Rule: if the bundling works, change only what's strictly necessary. The `.so` files go in as-is; only the `.dylib` files and the `whisper-cli` binary get rpath surgery.

## Known issues / watch out

- `AVAudioEngine.inputNode.outputFormat` must only be queried after mic permission is granted ‚Äî crash if called before.
- `booster.removeTap(onBus: 0)` must be called before `engine.stop()` ‚Äî occasional EXC_BAD_ACCESS on rapid start/stop otherwise.
- `DYLD_LIBRARY_PATH` is set on the child Process environment; this works for non-system binaries but may be stripped by SIP in edge cases.
- Whisper `--prompt` context is not yet wired to previous transcripts in the UI.
- Model validation (`validateModelWorks`) runs whisper-cli sequentially per file; validating a Large V3 model can take 30‚Äì60 s ‚Äî the spinner will show for that duration.
- `transcribe()` calls `proc.waitUntilExit()` directly; for large models this briefly ties up a Swift concurrency thread (not the main thread). Acceptable for now.
